SwarmHawk API v2

Programmatic access to attack surface scanning, domain risk scores, CVE/EPSS enrichment, and NIS2 compliance — from one REST API.

QUICK START

1. Get an API key

Sign in to your SwarmHawk account → Account tab → API Keys → Create key. Your key starts with swh_.

2. Run your first scan

curl -s https://api.swarmhawk.com/api/v2/scan \
  -H "X-API-Key: swh_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"domain": "example.com"}'

3. Parse the response

{
  "domain":     "example.com",
  "risk_score": 47,
  "critical":   1,
  "warnings":   3,
  "scanned_at": "2026-03-20T10:15:30Z",
  "checks": [
    {
      "check":        "ssl",
      "status":       "warning",
      "title":        "SSL expires in 18 days",
      "detail":       "Certificate CN=example.com expires 2026-04-07",
      "score_impact": 12
    },
    ...
  ]
}
AUTHENTICATION

X-API-Key header

All API v2 requests must include your key in the X-API-Key header. Keys are never stored in plaintext — only a SHA-256 hash is kept server-side.

X-API-Key: swh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Base URL

https://api.swarmhawk.com

All v2 endpoints are prefixed with /api/v2/. API v1 endpoints (/api/v1/scan) remain available for backward compatibility.

ENDPOINTS
SCANNING
POST /api/v2/scan Trigger a full domain scan ▼

Request body

{ "domain": "example.com" }

Response — returns risk score, finding count, and array of checks with CVE/EPSS enrichment and AI synthesis.

{
  "domain":     "example.com",
  "risk_score": 47,
  "critical":   1,
  "warnings":   3,
  "scanned_at": "2026-03-20T10:15:30Z",
  "checks": [ ... ]
}

Notes: Rate-limited by your API plan. Each call consumes 1 credit. Scan runs synchronously and returns when complete.

DOMAINS
GET /api/v2/domains List all monitored domains ▼

Returns all domains belonging to the authenticated account, with current risk score and last scan time.

Adds the domain and triggers the initial scan in the background. Returns the new domain record.

GET /api/v2/domains/{id}/history Historical scan results ▼

Returns up to 52 weekly scan records for trend analysis. Each record includes risk score, critical/warning counts, and timestamp.

{ "history": [ { "risk_score": 47, "critical": 1, "warnings": 3, "scanned_at": "..." } ] }
GET /api/v2/domains/{id}/nis2 NIS2 compliance status ▼

Returns NIS2 Directive compliance mapping for the domain. Maps security checks to NIS2 Article 21 obligations (incident handling, supply chain, access control, etc.).

{ "compliant": false, "score": 62, "articles": [ { "article": "21.2.b", "status": "fail", "detail": "..." } ] }
POST /api/v2/domains/{id}/rescan Trigger a fresh scan ▼

Queues a new scan for the domain. Returns immediately; scan runs in background. Poll /report to get results.

{ "status": "queued", "domain_id": "uuid" }
DELETE /api/v2/domains/{id} Remove a domain ▼

Removes the domain and all its scan history from your account. This action is irreversible.

{ "deleted": true }
API KEYS
POST /api/v2/keys Create an API key ▼

Generates a new API key for your account. The full key is returned once — store it securely. Subsequent calls return the existing key without the secret.

{ "created": true, "full_key": "swh_...", "keys": [ { "id": "...", "key_prefix": "swh_abc123" } ] }
GET /api/v2/keys List active API keys and usage ▼ 
{ "keys": [ { "id": "...", "key_prefix": "swh_abc123", "calls_this_month": 47, "limit_per_month": 500 } ] }
POST /api/v2/keys/{id}/regenerate Rotate a key ▼

Revokes the existing key and issues a new one, preserving the same usage limit. Useful for key rotation policies.

DELETE /api/v2/keys/{id} Revoke a key ▼

Permanently revokes the key. Any subsequent requests using this key will return 401.

KNOWLEDGE BASE — FREQUENTLY ASKED QUESTIONS
What is the difference between API v1 and API v2? ▼
API v2 is the clean public developer API containing only scan, domain, key, and account endpoints. API v1 (/api/v1/scan, /api/v1/keys) remains available for backward compatibility but we recommend migrating to v2. Admin, marketing, and outreach endpoints are not part of the public API and are not accessible via API keys.
How does the 3-layer validation pipeline work? ▼
Every Nuclei finding goes through three validation layers before appearing in the report:

L1 — Syntactic: Evidence must be populated, matched URL must be present, response body must be non-empty.
L2 — Template quality: Only confirmed or high-confidence Nuclei templates pass.
L3 — NVD/EPSS cross-reference: Critical/High findings require ≥0.70 confidence score. Medium+ requires ≥0.50. EPSS exploitation probability is surfaced alongside CVSS.

This reduces false positive rates by ~60–80% compared to raw Nuclei output.
What does the tamper-evident audit log do? ▼
Every agent action during a scan is recorded to a local SQLite database with SHA-256 hash chaining — each entry's hash depends on all previous entries. Any post-hoc modification breaks the chain. Run swarmhawk audit (CLI) to verify chain integrity. This makes reports defensible in legal or compliance contexts.
Can I use the API for automated CI/CD security scanning? ▼
Yes. Use the GitHub Action or call the API directly in your pipeline. The --mock flag (CLI) or mock mode (coming to API) runs the full pipeline with synthetic data in ~10 seconds — suitable for PR checks with no external calls. Set a risk score threshold as your pass/fail gate. 
# GitHub Actions example
- name: SwarmHawk Security Scan
  uses: hastikdan/swarmhawk-cli@v1
  with:
    target: ${{ env.DOMAIN }}
    api-key: ${{ secrets.SWARMHAWK_API_KEY }}
    fail-on: critical   # or: high, medium
Is there a rate limit per request, or just monthly? ▼
Rate limits are monthly per API key. There is no per-second or per-minute throttle at this time. Monthly counters reset on the 1st of each month. You can check current usage at GET /api/v2/plan.
What data is NOT accessible via the public API? ▼
The public API (v2) exposes only your own scan data, domain results, and account information. The following are not accessible via API keys and require admin authentication:

• Platform-wide user list and admin stats
• Other users' domains or scan history
• Outreach and marketing prospect data
• Platform revenue or LLM cost metrics
• Email template management
What AI model generates the scan summaries? ▼
AI synthesis is powered by Claude (Anthropic) or OpenAI GPT-4 via Portkey AI gateway for routing and cost control. The prompt includes validated findings, CVSS/EPSS scores, and domain context. Output includes: business impact assessment, recommended remediation steps, and exploitability likelihood. You can supply your own OpenAI or Anthropic API key via the CLI (--ai-key) or configure it in the cloud platform.
How do I report a security vulnerability in SwarmHawk itself? ▼
Do not open a public GitHub issue. Email security@swarmhawk.com with subject [SECURITY] <brief description>. We acknowledge within 48 hours and target a fix within 14 days for critical issues. Full policy at SECURITY.md.
